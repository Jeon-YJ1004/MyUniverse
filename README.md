# 🌌 MyUniverse — 3D Interactive Portfolio

> **인터랙티브 3D 은하수 궤도를 통해 프론트엔드 엔지니어의 우주(기술 스택, 프로젝트, 실무 경험)를 여행하는 포트폴리오 웹사이트입니다.**
> 
> 🔗 **배포 주소**: [https://yjjeon.cloud](https://yjjeon.cloud)

---

## 🛠 Tech Stack

* **Core**: React 19, TypeScript
* **3D Graphics**: Three.js
* **Build tool**: Vite (Rollup 기반 번들링)
* **Hosting**: Vercel (https://yjjeon.cloud)

---

## 🎨 주요 구현 기능

### 1. WebGL 기반 3D 입체 은하수 홈화면
* **Three.js**를 활용하여 중앙 행성, 고리, 450개의 미세 별빛 파티클 소행성대로 구성된 행성계를 구축했습니다.
* 마우스의 미세한 움직임을 감지하여 화면 시점이 부드럽게 기울어지는 **3D 패럴랙스 카메라** 기법을 연동했습니다.
* 각 서브페이지 위성(Profile, Projects, Experience, Contact)과 핵심 기술 스택(`React`, `TypeScript`, `Next.js` 등)이 각각의 입체 경사 궤도를 따라 독립적인 속도로 실시간 공전(Orbiting)합니다.

### 2. 가독성을 위한 하이브리드 투영 아키텍처
* 3D 그래픽스 상에 2D 텍스트가 렌더링될 때 깨지는 현상을 극복하기 위해, **3D 가상 객체의 좌표를 브라우저 2D 스크린 좌표로 투영(Projecting)**하여 HTML 위성 카드와 맵핑했습니다.
* 카메라 깊이(Z-Buffer) 연산값에 비례하여 위성 카드의 스케일, 불투명도(Opacity), `z-index`를 동적으로 제어함으로써 입체적인 차폐 효과와 높은 가독성을 동시에 만족시킵니다.

### 3. 반응형 3D 카메라 줌 보정
* 화면 종횡비(Aspect Ratio)와 너비를 실시간으로 계산하여, 모바일/태블릿 등 좁은 화면으로 진입할 때 카메라를 뒤로 줌아웃(Zoom-out)시키는 삼각함수 공식 `targetZ = Math.max(30.0 / aspect, 28.0)`을 구현하여 어느 기기에서도 궤도가 잘리지 않고 중앙에 안착되도록 설계했습니다.

## 🚨 개발 중 마주한 기술적 문제와 해결 과정 (Troubleshooting)

### 1. Node.js 버전 호환성에 따른 Vite 8 빌드 오류 (`Rolldown` 바이너리 누락)
* **문제 상황**: 
  - 개발 환경의 Node.js v20.13.1 버전에서 최신 Vite 8로 빌드 시도 시, 내장 컴파일 엔진인 `Rolldown` 바이너리 누락 오류가 발생하며 빌드가 비정상 종료됨.
* **해결 방법**: 
  - 아직 특정 윈도우 Node.js 아키텍처에서 빌드 안정성이 완벽히 확보되지 않은 최신 버전 대신, Rollup 기반으로 안정적으로 동작하는 **Vite v6 버전으로 패키지를 지정 다운그레이드**하여 컴파일러 호환성 문제를 완벽히 우회했습니다.

### 2. 윈도우 OS 에셋 파일 잠김(File Lock)에 의한 빌드 삭제 거부 오류
* **문제 상황**: 
  - 빌드 시 윈도우 백그라운드 프로세스가 기존 아웃풋 폴더인 `dist` 내 파일 핸들을 붙잡아 두고 있어, 컴파일러가 해당 폴더를 삭제하거나 새로 쓰기를 시도할 때 권한 거부(File Locked) 오류가 발생하여 빌드가 빈번히 중단됨.
* **해결 방법**: 
  - `vite.config.ts` 및 Vercel 배포 설정의 빌드 타겟 출력 경로를 기본값 `dist`에서 **`dist_build`**로 우회 설정함으로써, 프로세스 리소스 충돌 현상을 원천 방지하고 컴파일 안전성을 극대화했습니다.

### 3. Three.js `THREE.Clock` Deprecation 콘솔 경고 해결
* **문제 상황**: 
  - 3D 프레임 루프 렌더링에 사용하던 `THREE.Clock` 모듈에 대해 브라우저 콘솔에서 *"This module has been deprecated. Please use THREE.Timer instead."* 라는 경고 메시지가 지속 노출되어 클라이언트 콘솔을 지저분하게 만듦.
* **해결 방법**: 
  - 최신 릴리즈 규격에 부합하도록 애니메이션 루프의 타이밍 라이브러리를 **`THREE.Timer`** API로 리팩토링했습니다. TypeScript 컴파일 시 타입 유실 에러를 피하기 위해 `new (THREE as any).Timer()` 형태로 타입 캐스팅을 가미하고 `timer.update()`, `timer.getElapsed()` 체계로 시간 계산 로직을 완전 개편하여 경고를 깨끗하게 지웠습니다.

### 4. 화면 축소/모바일 해상도 진입 시 외곽 궤도 잘림 현상
* **문제 상황**: 
  - 데스크톱 가로 모드보다 세로 비율이 긴 모바일 기기나 브라우저 창을 극단적으로 줄였을 때, 고정된 카메라 Z축 거리(`14.5`) 때문에 가장 바깥쪽 궤도와 위성 카드가 뷰포트 영역 밖으로 밀려나 클릭이 불가능해지는 버그가 발견됨.
* **해결 방법**: 
  - 원근 투영 카메라의 세로 시야각(Vertical FOV = 50도) 삼각함수 공식에 입각하여, 최외곽 궤도 지름과 패널 마진을 포함한 안전 가시 영역 너비가 뷰포트 내에 100% 들어오기 위한 Z축 거리 공식을 수학적으로 유도했습니다:
    $$\text{targetZ} = \max\left( \frac{30.0}{\text{aspect}}, 28.0 \right)$$
  - 이를 `handleResize` 리사이즈 감지 시마다 반영하고 루프 내에서 카메라 Z축 거리를 부드럽게 보간(Lerp) 처리하여, 어떤 디바이스 환경에서도 은하계 전체가 안전하게 중앙 정렬되도록 반응형 배율을 완성했습니다.

### 5. 다크 모드 및 소행성 텍스트의 가시성/명암 대비 취약 문제
* **문제 상황**: 
  - 어두운 성운/우주배경 속에서 궤도를 도는 기술 스택 글자가 회색빛으로 묻히고, 마우스 오버 시에만 나타나는 위성 카드 텍스트가 식별하기 어려워 UX 편의성이 떨어짐.
* **해결 방법**: 
  - 위성 카드의 원형 아이콘 크기를 기존 `44px`에서 **`54px`**로 확대하고, 다크 모드 메인 텍스트 컬러(`--text`)를 미색에서 **완전한 순수 화이트(`#ffffff`)**로 상향했습니다.
  - 보조 텍스트 컬러(`--muted`)의 불투명도도 `0.38`에서 **`0.58`**로 뚜렷하게 다듬었으며, 각 기술 스택의 소행성 글자들마다 **상징적인 고유 브랜드 컬러(React: 하늘색 등)**와 텍스트 백글로우 드롭섀도를 주어 시인성을 극적으로 확보했습니다.

## ⚙️ 실행 및 빌드 방법

### 1. 의존성 패키지 설치
```bash
npm install
```

### 2. 로컬 개발 서버 구동
```bash
npm run dev
```

### 3. 프로덕션 빌드 (출력 경로: `dist_build`)
```bash
npm run build
```
